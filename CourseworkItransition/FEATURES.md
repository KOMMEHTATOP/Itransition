# Inventory Management — Описание функционала

## Что это

Веб-приложение для создания произвольных коллекций (инвентарей) с настраиваемыми полями.
Пользователь создаёт инвентарь — задаёт его структуру (поля). Другие пользователи заполняют его элементами (айтемами).

**Пример:** инвентарь «Ноутбуки» с полями «Модель» (строка) и «Цена» (число). Другие добавляют свои ноутбуки.

---

## Роли и права доступа

| Кто | Что может |
|---|---|
| Гость (не залогинен) | Просматривать публичные инвентари, айтемы, поиск |
| Авторизованный пользователь | Создавать инвентари, добавлять айтемы в публичные инвентари, лайкать, комментировать |
| Владелец инвентаря | Полный контроль над своим инвентарём: поля, настройки, доступ, удаление |
| Пользователь с явным доступом | Добавлять/редактировать/удалять айтемы в приватном инвентаре |
| Admin | Всё то же что владелец для любого инвентаря; управление пользователями |

---

## Авторизация

- **Регистрация / вход** по email и паролю — `/register`, `/login`
- **OAuth**: Google и GitHub (только на продакшне)
- Токен хранится в `localStorage`, добавляется в каждый запрос автоматически
- Заблокированный пользователь не может войти — сервер возвращает 403

---

## Главная страница `/`

- Таблица **последних 10 публичных инвентарей** (сортировка по дате)
- Таблица **топ-5 инвентарей** по количеству элементов
- **Облако тегов** — размер слова пропорционален частоте; клик → поиск по тегу
- Доступно без авторизации

---

## Список инвентарей `/inventories`

- Таблица всех публичных инвентарей с пагинацией (20 штук на страницу)
- Сортировка: по дате (новые/старые), по названию
- Клик на строку → страница инвентаря
- Кнопка **«+ Create»** (только авторизованным) → модальное окно создания
- Чекбоксы + кнопка **«Delete selected»** в тулбаре (только для своих)

---

## Страница инвентаря `/inventories/:id`

Шесть вкладок:

### Items (Элементы)
- Таблица элементов: Custom ID + поля с `ShowInTable=true` + дата + автор
- Клик на строку → страница элемента `/items/:id`
- Тулбар: **«+ Add item»** → модальное окно, **«Delete selected»**
- Пагинация и сортировка
- Доступно для чтения всем

### Discussion (Обсуждение)
- Линейный чат с поддержкой **Markdown** в сообщениях
- Новые комментарии появляются в реальном времени через **SignalR** (без перезагрузки)
- Писать могут только авторизованные пользователи

### Statistics (Статистика) — только чтение
- Общее количество элементов
- Для **числовых полей**: мин, макс, среднее, кол-во заполненных
- Для **текстовых полей** (Text, MultilineText, Link): топ-5 самых частых значений

### Fields (Поля) — только владелец/admin
- Список кастомных полей с drag-and-drop сортировкой
- Типы полей: `Text`, `MultilineText`, `Number`, `Link`, `Boolean`
- Лимит: до **3 полей каждого типа**
- Для каждого поля: название, описание, чекбокс «Показывать в таблице»
- Кнопка ✕ удаляет поле (и все значения айтемов для него)

### Custom ID (Кастомный ID) — только владелец/admin
- Конструктор формата автогенерируемого ID — drag-and-drop, до 10 элементов
- Типы элементов:

| Тип | Описание | Формат |
|---|---|---|
| Fixed | Фиксированный текст, emoji | — |
| Random20bit | Случайное число 0–1 048 575 | `X5` = 5 hex, `D6` = 6 десятичных |
| Random32bit | Случайное число 0–2 147 483 647 | `X8` = 8 hex |
| Random6digit | Случайное 6-значное число | — |
| Random9digit | Случайное 9-значное число | — |
| GUID | UUID v4 | — |
| DateTime | Дата/время создания UTC | `yyyyMMdd`, `HH`, `mm` и т.д. |
| Sequence | Порядковый номер, уникальный для инвентаря | `D3` = 3 цифры с ведущими нулями |

- Кнопка **↺** генерирует превью прямо на странице
- Если при создании айтема Custom ID оставлен пустым — генерируется автоматически
- Изменение формата не меняет существующие ID

### Settings (Настройки) — только владелец/admin
- **Название** инвентаря
- **Описание** — WYSIWYG Markdown-редактор (`@uiw/react-md-editor`)
- **Категория** — выбор из списка
- **Видимость** — публичный / приватный
- **Обложка** — drag-and-drop загрузка изображения через Cloudinary (URL в БД)
- **Теги** — мультитег с автодополнением из существующих тегов
- **Автосохранение** каждые 8 секунд — индикатор «⏳ Saving…» / «✓ Saved»
- **Optimistic locking**: при конфликте (кто-то сохранил раньше) — предупреждение «⚠ Conflict»

### Access (Доступ) — только владелец/admin
- Показывает список пользователей с явным доступом
- Поиск пользователя по имени или email с автодополнением
- Добавить / удалить пользователей
- Сортировка списка по имени или email

---

## Страница элемента `/items/:id`

- Все поля элемента в режиме редактирования (авторизованный с доступом) или просмотра
- **Custom ID** — редактируемое поле
- **Автосохранение** + optimistic locking (аналогично инвентарю)
- **Лайки**: кнопка лайка с счётчиком (один пользователь — один лайк)
- Кнопка **Delete** в тулбаре (не в строке)

---

## Личный кабинет `/profile`

- Таблица **«Мои инвентари»** — свои инвентари с пагинацией и сортировкой
- Таблица **«Accessible to Me»** — публичные и инвентари с явным доступом (чужие)
- Создание нового инвентаря
- Удаление нескольких инвентарей через чекбоксы

---

## Поиск `/search`

- Глобальный поиск в шапке (debounce 300 мс) — доступен всем включая гостей
- **Full-text search** через PostgreSQL `tsvector` — по названию и описанию инвентарей, по значениям полей айтемов
- Результаты в двух таблицах: «Inventories» и «Items»
- Поиск по тегу: клик на тег в облаке → `/search?tag=...`

---

## Панель администратора `/admin`

Доступна только пользователям с ролью **Admin**.

- Таблица всех пользователей: имя, email, статус (Active / Blocked), роль (Admin), кол-во коллекций, дата регистрации
- Тулбар сверху (действия применяются к выбранным через чекбоксы):
  - **Block** — заблокировать (нельзя заблокировать себя)
  - **Unblock** — разблокировать
  - **Make Admin** — дать роль Admin
  - **Remove Admin** — снять роль Admin (можно снять у себя)
  - **Delete** — удалить пользователей (нельзя удалить себя)
- Кнопка Admin в navbar видна только администраторам

## UI / UX

- **Два языка**: английский и русский — переключатель EN/RU в navbar, выбор сохраняется в `localStorage`
- **Две темы**: светлая и тёмная — переключатель ☀️/🌙 в navbar, выбор сохраняется в `localStorage`
- Responsive: Bootstrap 5, мобильное меню через burger-кнопку
- Все таблицы — `table`-представление (без gallery/tiles)
- Нет кнопок Edit/Delete в строках таблиц — только тулбар сверху с чекбоксами
